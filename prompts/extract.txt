以下は「{domain}」分野のClaude Code会話群から抽出した要約である。
各会話にはfiles(操作されたファイルパス)とcmds(実行されたコマンド)のメタデータが含まれる。

**この分野でのみ必要とされるドメイン固有の知識**を抽出せよ。
具体的には: この分野の経験者だけが知っている判断基準、設計上の落とし穴、ドメイン特有のワークフロー。

除外すべきもの（ドメイン固有ではない）:
- サブLLMによるコード検証ループ（claude -p で別AIを呼んで差分検証する手順）
- cargo build / cargo test / cargo check などのビルド・テストサイクル
- git status / git diff / git commit などのバージョン管理操作
- ファイル読み込み→編集→保存の汎用的なコード開発手順
- 「検証結果だけ返せ」のようなCLAUDE.md由来の運用指示

制約:
- 最大5パターンまで。
- 抽出基準は「回数」ではなく「そのトピックについてユーザーとの対話（やり取り）があったかどうか」。
  - ユーザーがAIの出力を訂正した（「〜だぞ」「違う」「〜じゃない」等）
  - ユーザーがAIに判断基準を教えた（「和暦だぞ」「期限は1年後だ」等）
  - 同じトピックでユーザーとAIが複数ターン往復した
  これらは1回の会話でも重要なパターンとして抽出せよ。
- 汎用的な開発手順をドメイン知識と混同するな。「cargo buildで検証」はどの分野でもやること。

JSON配列で返せ。各要素:
{{
  "title": "パターン名",
  "description": "何をするパターンか（なぜこの分野で重要なのか含む）",
  "steps": ["手順1", "手順2"],
  "discussed": true
}}
